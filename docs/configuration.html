Architecture

Configuration

Firn is configured entirely through environment variables. Either FIRNFLOW_STORAGE_URI (preferred) or the legacy FIRNFLOW_S3_BUCKET is required; everything else has a sensible default.

Core settings

VariableDefaultDescription
FIRNFLOW_STORAGE_URI required (or set FIRNFLOW_S3_BUCKET) Object-storage root that every namespace lives under. Accepts a URI of the form scheme://bucket[/prefix...] where scheme is s3 (any S3-compatible backend) or gs (native Google Cloud Storage). Namespace tables live at {root}/{namespace}/. Multi-segment prefixes are supported, e.g. s3://shared-bucket/tenants/acme/prod or gs://shared-bucket/tenants/acme/prod.
FIRNFLOW_S3_BUCKET legacy fallback Legacy S3-only form. Equivalent to FIRNFLOW_STORAGE_URI=s3://<bucket>. Both vars remain supported; if both are set they must agree, or startup fails.
FIRNFLOW_BIND 0.0.0.0:3000 Address and port for the HTTP listener. Use 127.0.0.1:3000 to restrict to localhost.
FIRNFLOW_MAX_BODY_BYTES 16777216 (16 MB) Maximum request body size for every JSON endpoint. Raise this when upserting large batches (multivector workloads in particular hit the ceiling fast, around 2 MB per row at 512 sub-vectors of dimension 128). Lower it to tighten the DoS surface in front of a trusted client.
RUST_LOG info Log level filter. Supports tracing directives like firnflow_api=debug,firnflow_core=trace.

Object storage backend

For S3-family backends, the FIRNFLOW_S3_* variables below configure the endpoint and credentials. For native Google Cloud Storage, set FIRNFLOW_STORAGE_URI=gs://<bucket> and one of the standard GOOGLE_* credential variables — no other firnflow-specific configuration is required.

S3-family variables

For real AWS S3, you typically only need FIRNFLOW_STORAGE_URI and FIRNFLOW_S3_REGION. Credentials come from the standard AWS chain (instance profile, environment, ~/.aws/credentials).

VariableDefaultDescription
FIRNFLOW_S3_ENDPOINT unset (use AWS) Custom S3 endpoint URL. Set this for MinIO, R2, Tigris, DigitalOcean Spaces, or any S3-compatible backend. When set, path-style addressing and plain HTTP are automatically enabled. Not used for native GCS — set FIRNFLOW_STORAGE_URI=gs://... instead.
FIRNFLOW_S3_ACCESS_KEY unset S3 access key ID. Falls back to the standard AWS credential chain if not set.
FIRNFLOW_S3_SECRET_KEY unset S3 secret access key. Falls back to the standard AWS credential chain if not set.
FIRNFLOW_S3_REGION us-east-1 Region for the bucket. Some S3-compatible providers (R2, Tigris) accept auto.

Google Cloud Storage variables

Native GCS routing uses lancedb's gcs feature and the matching object_store::gcp client. Both authenticate from the standard Google environment variables; firnflow forwards them unmodified.

VariableDefaultDescription
GOOGLE_APPLICATION_CREDENTIALS unset Path to an application-default-credentials file. May resolve to a service-account JSON, federated identity, or user creds.
GOOGLE_SERVICE_ACCOUNT_PATH unset Path to a service-account JSON file. Strictly service-account format only — distinct from GOOGLE_APPLICATION_CREDENTIALS.
GOOGLE_SERVICE_ACCOUNT_KEY unset Inline service-account JSON, as a single string. Useful for environments where mounting a file is awkward (some serverless platforms). Redacted from Debug output.

AWS S3

For production AWS deployments, let the standard credential chain handle authentication (instance profiles, ECS task roles, or ~/.aws/credentials).

FIRNFLOW_STORAGE_URI=s3://my-firn-bucket
FIRNFLOW_S3_REGION=eu-west-1

FIRNFLOW_S3_ACCESS_KEY=<r2-access-key>
FIRNFLOW_S3_SECRET_KEY=<r2-secret-key>
FIRNFLOW_S3_REGION=auto

FIRNFLOW_S3_ENDPOINT=https://t3.storage.dev
FIRNFLOW_S3_ACCESS_KEY=<tigris-access-key>
FIRNFLOW_S3_SECRET_KEY=<tigris-secret-key>
FIRNFLOW_S3_REGION=auto

DigitalOcean Spaces

FIRNFLOW_STORAGE_URI=s3://firn-spaces
FIRNFLOW_S3_ENDPOINT=https://<region>.digitaloceanspaces.com
FIRNFLOW_S3_ACCESS_KEY=<spaces-access-key>
FIRNFLOW_S3_SECRET_KEY=<spaces-secret-key>
FIRNFLOW_S3_REGION=<region>

Google Cloud Storage (native)

Use the gs:// scheme. The GCS S3-interop endpoint is a separate path that silently drops If-None-Match: * and is not supported — do not reach for it.

FIRNFLOW_STORAGE_URI=gs://my-firn-bucket
GOOGLE_APPLICATION_CREDENTIALS=/etc/firnflow/gcp-sa.json
Provider validation
Firn is validated against AWS S3, MinIO, Cloudflare R2, Tigris, DigitalOcean Spaces, and native Google Cloud Storage with 100-run concurrent-writer stress tests. Backblaze B2 and the GCS S3-interop endpoint are not supported. See the compatibility matrix in the project README for the per-provider details.

Cache settings

The foyer hybrid cache has two tiers: a RAM tier for the hottest data and an NVMe tier for overflow. Both tiers are shared across all namespaces.

VariableDefaultDescription
FIRNFLOW_CACHE_MEMORY_BYTES 67108864 (64 MB) RAM tier capacity in bytes. This is the L1 cache; hits here are sub-microsecond.
FIRNFLOW_CACHE_NVME_PATH /tmp/firnflow-cache Directory for the foyer NVMe-tier block file. In production, point this at a fast local SSD. The Docker image defaults to /var/lib/firnflow/cache.
FIRNFLOW_CACHE_NVME_BYTES 268435456 (256 MB) NVMe tier capacity in bytes. This is the L2 cache; hits here avoid the object-storage round-trip but are slower than RAM.

Cache sizing guidance

The right cache size depends on your working set. A cached result for a 10-result query with 1536-dim vectors is roughly 62 KB. Some rules of thumb:

Working setRAM tierNVMe tier
Development / testing64 MB256 MB
Small production (hundreds of cached queries)256 MB1 GB
Large production (thousands of cached queries)1 GB10 GB
Cache is ephemeral
The cache is a performance optimisation, not durable storage. Data in the configured object store is always the source of truth. If the cache is lost (restart, crash, disk failure), Firn repopulates it transparently on the next query.

Object cache

The object cache is a separate, optional layer from the result cache above. It is a local byte-range cache that sits under the storage engine: on a cache miss the engine still reads the namespace's data and index files from object storage, but those reads are served from local disk on subsequent queries. It accelerates cold and novel queries (different filters, new vectors) that the result cache cannot help, because they never hit object storage twice for the same bytes.

It only caches immutable, uniquely-named objects (data fragments and index files, which are written once and never mutated). Manifests, version pointers, and transaction logs are always read straight through, so a write, delete, compaction, or index build is reflected immediately — the cache cannot serve stale data. It is disabled by default.

VariableDefaultDescription
FIRNFLOW_OBJECT_CACHE_ENABLED false Enable the byte-range object cache. When off, every object-storage read goes straight to the backend.
FIRNFLOW_OBJECT_CACHE_DIR /tmp/firnflow-object-cache Directory for cached object bytes. In production, point this at a fast local SSD/NVMe. Existing files are scanned and counted toward the capacity on startup, so the cache survives a restart within its byte budget.
FIRNFLOW_OBJECT_CACHE_BYTES 10737418240 (10 GB) Maximum bytes of cached object data held on disk. When full, least-recently-used entries are evicted. The cap is enforced across restarts (small per-entry size sidecars are negligible additional overhead).
FIRNFLOW_OBJECT_CACHE_MAX_ENTRY_BYTES 268435456 (256 MB) Largest single read the cache will buffer and store. A read above this is streamed straight through and not cached, so one large fragment read cannot spike memory by its full size. Lower it on memory-constrained hosts; raise it to cache larger fragments whole.
Result cache vs object cache
The result cache stores whole query results and turns a repeated query into zero object-storage reads. The object cache stores raw file byte ranges and makes the first execution of a new query cheaper the next time similar data is read. They are complementary and can be enabled together.

Authentication

Firn supports optional bearer-token authentication. Both keys are opt-in via env: when neither is set, the API stays open and logs a single startup WARN (the default for the local-dev compose stack).

VariableDefaultDescription
FIRNFLOW_API_KEY unset (open) Read/write tier. Authorises upsert, query, list, and warmup. Send as Authorization: Bearer <token>. Compared in constant time.
FIRNFLOW_ADMIN_API_KEY unset Admin tier. Authorises destructive ops (delete, index, fts-index, scalar-index, compact) and read/write ops. If unset, FIRNFLOW_API_KEY authorises admin routes too (single-key fallback).
FIRNFLOW_METRICS_TOKEN unset (open) Optional gate on /metrics. Same Bearer parser as the data plane, so Prometheus's bearer_token / bearer_token_file scrape config works directly. /metrics stays public when this is unset.

Status codes:

  • 401 Unauthorized — missing, malformed, or unknown token. Response includes WWW-Authenticate: Bearer realm="firnflow".
  • 403 Forbidden — valid token but insufficient scope (write key on an admin route while a separate admin key is configured).
Service-level only
This authenticates clients to the firnflow process. It does not bind a credential to a namespace: any holder of FIRNFLOW_API_KEY can read or write any namespace. For per-tenant namespace isolation, place Firn behind an authenticating gateway that maps tenants to namespaces.

Rate limiting

Two layers, both off by default. The per-principal limiter buckets on the validated identity attached by the auth layer, so a bogus token cannot mint fresh buckets. The optional pre-auth limiter sits outside auth and caps unauthenticated traffic by peer IP.

VariableDefaultDescription
FIRNFLOW_RATE_LIMIT_RPS unset (disabled) Sustained requests per second per principal. 0 or unset disables the per-principal limiter.
FIRNFLOW_RATE_LIMIT_BURST 30 Token-bucket burst size. Used by both limiters when active.
FIRNFLOW_PREAUTH_IP_LIMIT_RPS unset (disabled) Optional pre-auth IP limiter. When set, a coarse limiter wraps the protected sub-router as the outermost layer and caps credential-stuffing throughput per peer IP.
FIRNFLOW_TRUST_PROXY_HEADERS false When true, the IP extractor reads peer IP from X-Forwarded-For / X-Real-IP. Default reads from the connection only — safe for deployments reachable directly. Enable only when Firn sits behind a trusted proxy that strips the client-supplied header.

/health and /metrics are exempt from both limiters. Rejected requests return 429 Too Many Requests with a Retry-After header.

Docker Compose override

To customise settings for local development, create a docker-compose.override.yml:

# docker-compose.override.yml
services:
  firnflow:
    environment:
      FIRNFLOW_CACHE_MEMORY_BYTES: "268435456"   # 256 MB RAM cache
      FIRNFLOW_CACHE_NVME_BYTES: "1073741824"    # 1 GB NVMe cache
      RUST_LOG: "firnflow_core=debug,firnflow_api=debug"

Full example

A complete production configuration against AWS S3:

# Production environment
FIRNFLOW_STORAGE_URI=s3://prod-firn-vectors
FIRNFLOW_S3_REGION=eu-west-1
FIRNFLOW_BIND=0.0.0.0:3000
FIRNFLOW_MAX_BODY_BYTES=33554432            # 32 MB body limit for larger batches
FIRNFLOW_CACHE_MEMORY_BYTES=268435456
FIRNFLOW_CACHE_NVME_PATH=/var/lib/firnflow/cache
FIRNFLOW_CACHE_NVME_BYTES=10737418240
RUST_LOG=info
