# Engine RFCs

Design proposals for **hev search** (the engine behind hev layer; a hard fork of
firnflow developed as ours). These are **engine-scoped** â€” search, storage,
indexing, caching, metadata, filter, facets, tokenization. The *edge* view (how
Layer operates this engine) lives in `../../../layer/docs/rfcs/` â€” RFC 0086 there
is the Layer-side twin of this engine.

This series is numbered **independently** of Layer's series; an engine `0001` and
a Layer `0001` are unrelated. Apply the engine/edge test (`AGENTS.md`) before
adding one: if it's edge (auth, tenancy, rate limiting), it belongs in `../layer`,
not here.

| [0007](0007-point-fetch-and-nearest-to-id.md) | Point-fetch by id and nearest-to-stored-id query | Draft. **Additive, read-path only.** The engine can ANN on a caller-supplied vector but not on the vector of a row it already stores â€” there is no `nearest_to_id`, so Layer's `recommend` / "more like this" (`shop`) round-trips the seed vector out and back. The headline is the vector search, which the cache cannot serve. |
| [0008](0008-bulk-ingest-upsert-and-pipeline.md) | Bulk ingest â€” upsert, import, and the Layer Pipeline write path | Draft, exploration (options kept open). Frames the write-path design space: idempotent `/upsert` (JSON, latest-write-wins) vs bulk `/import` (Arrow IPC, insert-only, async). Catalogs the options (Pipeline drives `/import` directly, a trait `bulk_write`, unify import under upsert semantics, or accept JSON) and their constraints. Engine half; orchestration is Layer's Pipeline (RFC 0086). Gated on RFC 0005 (string ids) for the real workload. |
| [0009](0009-pluggable-vector-index-type.md) | Pluggable vector index type (HNSW) | Draft. **Additive.** The vector index is hardcoded to IVF_PQ (`create_index` always builds `Index::IvfPq`; the handler rejects any `kind != "ivf_pq"`), and the query path exposes only `nprobes` â€” no candidate re-ranking. Single-vector only in v1; SPFresh framed as a separate "outgrow LanceDB" follow-up. Engine-local. |

Adopts **Iso-ModernColBERT** (128-dim, cosine, f16, Apache-2.0 â€” matches the forced-cosine multivector constraint) as the reference model the BEIR multivector harness benchmarks against (incumbent baseline: `lightonai/LateOn` f32). int8/4-bit PLAID named as the next rung; pooled-FDE two-stage framed as a separate decision (the engine does MaxSim natively, unlike Turbopuffer in RFC 0049). Edge twin: Layer RFC 0089. Engine-local. |

The index-variant sweep is gated on RFC 0009; the store-vs-store / Turbopuffer comparison is the anonymized edge twin in `../layer` (Â§2.4), reusing this harness's loaders + ground-truth. | [0012](0012-lsn-read-consistency.md) | LSN-based read consistency | Draft, proposal. **Additive read-path + write-response change.** Layer's consistency watermark is a Turbopuffer-shaped timestamp (`_hevlayer_upserted_at`) that the engine **strips on write** (`clients/search.rs:355â€“361`), so the gateway's injected stable-read predicate is a silent no-op that fails open â€” folded here as the motivating defect. Answers RFC 0008's bulk-consistency question; touches RFC 0011's freshness axis. Engine-local. |
